\subsection{Networking:}

PHOTO of Networking
\begin{itemize}
    \item Networking refers to the practice of connecting computers, devices, and systems to share resources and communicate with each other. It enables the transfer of data and information across various platforms, both within local environments (like a home or office) and globally (like the internet).
    \item \textbf{Local Area Network (LAN):} A network that connects devices within a limited area, such as a building or campus, allowing for the sharing of resources like printers and files.
\item \textbf{Wide Area Network (WAN):} A network that covers a larger geographical area, such as cities, countries, or even continents, connecting LANs and allowing global communication and data transfer .
\item \textbf{Wireless Networks:} Networks that do not require physical cables to connect devices, such as Wi-Fi, which uses radio waves to transmit data.
\item \textbf{Internet:} A global network of networks that connects millions of private, public, academic, business, and government devices, enabling communication and access to a vast array of services and information.
\item \textbf{5G Networks:} The fifth generation of mobile network technology, offering significantly higher speeds, lower latency, and more reliable connections for mobile devices and IoT (Internet of Things).
\item \textbf{Virtual Private Network (VPN):} A secure, private network that allows users to send data over the internet as if they were on a private network, providing encryption and privacy for online activities.




\end{itemize}

\section{Version Control: Git and GitHub:}
    \subsection{Definition of Git :}
   \paragraph{  Git is a "distributed version control system" (VCS) that allows developers to track and manage changes to their code base, over time. It enables several programmers to work on the project together, by keeping history of changes to files; and supporting branching and merging so that the development team can go back to previous versions of the project if needed. Git allows each user to have a standalone copy of the entire repository instead of just the current version of files, unlike version control systems that are centralized .}
  \subsection{How Git Works :}  
  It organizes project file snapshots in a more incremental way. These snapshots preserve the state of the project at different moments in time. Here’s a rundown of the fundamentals :
  \subsubsection{Key Git Concepts :}
\begin{itemize}
\item Repository : a repositoty is a folder that contains all of the project files, the entire commit history, and all of the versioning data. It can be local or remote .
    \item Commit: Like a snapshot of the project at a particular point in time. Each commit includes :\newline
· The changes made.\newline
· A unique identifier (hash).\newline
· A commit message explaining the change.\newline
· Pointing to the parent commit (except for the first commit).\newline
\item Branch: You can create an independent branch to work with a version of the project. The primary branch is usually named main or master, but other branches can be created for new features, bug fixes, etc. This separation means that developers can work on features while the swagger for the stable version of the code is unaffected.
\item  Step 2 : Staging Area (Index): Files are added to the staging area before committing. This is where modifications are staged before committing them.
\item  Working Directory: This is the local directory where the actual files are stored. Files which have had changes made to them are not being tracked by Git until they are added to the staging area.
\end{itemize}
   \subsubsection{Git Workflow :  }                                                                    
 \paragraph{Here are the typical steps involved in your general workflow with Git:}
  1. Clone a Repository : If you are working on a pre-existing project, you clone the repository from some remote source (like GitHub) to your local machine using git clone .
  2. Push Changes to a Remote Repository:
Locally the changes are committed and then pushed to a remote repository (eg GitHub) using git push origin
  3. Fetching Changes from a Remote Repo:
To see if anyone else has changed the remote repository, use git pull origin .
\subsection{Importance of Git in Software Development : }Git is very vital in the current software development and the role of Git cannot be overemphasized. Here are the key reasons why Git is essential:
  \begin{itemize}
      \item  Collaboration : Git makes it possible for multiple developers to work on the same project at the same time without one developer’s changes affecting the others negatively. By means of branches every developer is able to develop new features as well as to fix bugs. Git makes it easier to deal with the merging of these changes to the master branch.developers can pull or merge requests to check and comment on the changes to be incorporated into the project.
      \item  Version Control : Everyone who uses Git can see the changes that have been made to the code- base over time. That is, if there is a mistake, one can go back to a previous version of the program or undo certain changes through the checkout command when using Git .This version control also enables developers to maintain different versions of the project (e.g.production, development, experimental) using different branches.
      \item Code Integrity and Quality : Git assists in preserving the historical accuracy of the project's source code by documenting all changes made to the source. If a bug is detected, Git can help a developer to track down the point in time that this bug was created and which amendments introduce it. This is achieved by evaluating various commits and examining what was altered between the two images as a result of the deviation.
\item Backup and Security : Every developer is able to retrieve deleted files as a result of Git 's distributed nature because every developer has a complete copy of the project on their machine.Moreover, the project can be stored on services such as GitHub or Git Lab and this is already an additional safe and off-site backup which prevents loss of important data.
      \item Branching and Feature Isolation : With Git comes the ability to branch out and work on separate bits and pieces of the main code so certain aspects can be isolated. New features or bugs can be added or fixed respectively without impacting the entire pre-existing code base. And once that new bit is complete, it can simply be snapped back to the main working area of the code.
      \item Continuous Integration and Deployment (CI/CD):When there is a constant flow of new codes being integrated with existing ones, Git serves as one of the key components of the process. When such sudden changes occur to a code repository, Git is the feature that runs a bunch of automated tests and pushes the amended codes. There are multiple apps such as GitHub Actions, Git Lab CI and Jenkins that when integrated with Git support automatic testing that in turn boosts productivity.

\end{itemize}
\section{ GitHub:}
\subsection{Definition of GitHub :}
  \paragraph{GitHub is a popular multi-faceted purpose of securing repositories, code collaboration, and open source projects contribution.A three-dimensional view of these three elements is as follows: }
  \subsubsection{Securing Repositories :}
\begin{enumerate}
    \item  Version Control 
 \paragraph{To start with, repositories on GitHub are hosted using Git, which is a distributed version control system. This effectively means that once a developer's code is kept in a repository, it will be possible to monitor and control any alterations made on that code, even to the extent of restoring previous versions if need be.}

 \item Private and Public Repositories :
\paragraph{Another type of repository that can be created on GitHub is Private repositories which are retained and viewable only by the owner of the repository and, some selected collaborators, and a public repository which any person can have access to .}

 \item Storage and Access: 
\paragraph{GitHub is said to be a cloud-based facility. Hence it is possible to access it from anywhere, making it easy to secure repository code and related documents, images amongst others.}

\end{enumerate}
  \subsubsection{Code Development :}   
\begin{itemize}
    \item Branches: Another structural form of protection of that code is through branches, whereby developers create branches or offshoots that allow them to create new features or fix bugs without impacting on the main code or the baseline version, which in most case is referred to as the main or master branch.
    \item Pull Requests (PRs): In a situation when there are no improvements, the developer then proposes merging that branch into the initial one. That begins the procedure in which other potential team members can offer their feedback with suggestions concerning that change as well as endorse (or disallow) the modification.
\item Issues and Projects: In addition, where developmental efforts are made, an issue tracking knitting helmet is integrated in GitHub.
\item Issues and Tasks: Issues Tracker in the Github Sites supports the issue tracking where the developers can post tasks, bugs or feature requests with the aim of creating new things. The Contributionssection essentially aids in the collations and monitoring of effort expended.
 \item Workflows and Integrations: With GitHub Actions users can automate several processes like CI/CD and many more directly in the platform.
 \item  Pull Requests: The process of code reviews is facilitated by GitHub where people in the same team review each other’s code before it is merged into the main branch. For specific changes made on the lines of code, inline comments can also be added.
 \item  Forking: Basically, to be able to participate in an open-source project, developers will have to fork the repository allowing them to make their own copy of the project. In case developers decide to make some changes, they will be able to send a contributing request afterwards allowing them to propose the changes to the parent repository.
 \item Contributing Guidelines: There are open-source projects which utilize the ‘CONTRIBUTING.md’ document to explain in details how contributors can participate in the project. This will most likely ensure uniformity and order in many contributions.
 \item  Star and Watch Repositories: The users ‘star’ repositories they wish to favorite and ‘watch’ them if they want to get notification regarding activities like issues and pullrequests that occur on the repository.
 \item Community and Discussion: GitHub provides the facilities for an active community, with discussion board options available for contributors and maintainers to discuss topics on the project.
 \item  Licensing: Most open-source projects on GitHub use widely spread licensing like MIT, GPL, or Apache, which specify what can be done with the code: use, modification, or redistribution.
\end{itemize}

\section{The widespread use of ICT (Information and Communication Technology) presents several new challenges in today's world:}

1. Privacy Problems: People worry about how their personal data is collected and used without permission.

2. Cyber-security Risks : Hacking, viruses, and online attacks can cause big problems for businesses and individuals.

3. Unequal Access : Not everyone has the same access to technology, especially in poor or rural areas.

4. Over-dependence:  If technology fails, many systems like banking, healthcare, or communication can stop working.

5. Fake News : False information spreads quickly online and confuses people.

6. Environmental Issues : Making and throwing away tech devices harms the environment and uses a lot of energy.

7. Mental Health : Spending too much time online can lead to stress, loneliness, or addiction.

8. Difficult to Control :Governments struggle to make rules to manage technology use safely.

9. Cultural Impact : Local cultures and languages might be lost because global platforms dominate.

These challenges mean we need to use ICT carefully and responsibly.

· The difference between ICT and  CS (computer science) : 
\subsection{ICT}
\begin{itemize}
\item Scope : ICT refers to the tools and technologies used to handle and disseminate information. It includes the infrastructure, devices, and systems used for communication.
\item Examples: Internet, smartphones, social media platforms, video conferencing tools, and broadcasting technologies.
\item Emphasis on communication, information sharing, and the integration of technology in fields like education, business, and media.
\item Objective: Facilitating communication and access to information for broader societal benefits.
\subsection{CS}
\item Scope: computer science focuses the study of computing systems, algorithms, programming, and data management.
\item Examples: Software development, database management, artificial intelligence, cybersecurity, and operating systems.
\item Developing and optimizing software and hardware solutions.
\item Objective: Solving technical problems and advancing computing technologies.
\subsection{Key Difference:}
\item \paragraph{ICT is more about using technology to enable communication and share information, while computer science is about creating and understanding the underlying technologies and systems that make such communication possible.}

\end{itemize}
